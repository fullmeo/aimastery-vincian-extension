## ğŸ—ï¸ Product Architecture

### Core Features

#### **Feature 1: Vincian Score**

**Description**: Objective 0-100 code beauty score based on 7 principles

**Principles**:

1. **Movement (Code Flow)**
   - Measures: Cyclomatic complexity, nesting depth, callback chains
   - Ideal: Linear flow, early returns, minimal branching
   - Example: Score 95 = simple, readable flow; Score 40 = nested spaghetti

2. **Balance (Symmetry)**
   - Measures: Variance in file/function sizes
   - Ideal: Consistent sizing across codebase
   - Example: Score 90 = all functions 5-25 LOC; Score 50 = mix of 5 LOC and 200 LOC

3. **Proportion (Right-Sizing)**
   - Measures: Average function/class size vs ideal ranges
   - Ideal: Functions 5-25 LOC, classes 50-300 LOC
   - Example: Score 85 = well-sized components; Score 45 = God classes

4. **Contrast (Clear Distinctions)**
   - Measures: Naming clarity, type annotations, layer boundaries
   - Ideal: Clear roles, strong typing, visible boundaries
   - Example: Score 100 = descriptive names + types; Score 30 = `a`, `b`, `tmp`

5. **Unity (Cohesion)**
   - Measures: LCOM4, coupling metrics, feature envy
   - Ideal: High cohesion, low coupling
   - Example: Score 90 = single responsibility; Score 40 = God class with unrelated methods

6. **Simplicity (Less is More)**
   - Measures: Abstraction layers, dead code, over-engineering
   - Ideal: As simple as possible, not simpler
   - Example: Score 95 = straightforward solution; Score 50 = unnecessary patterns

7. **Perspective (Architectural View)**
   - Measures: Documentation, circular deps, module structure
   - Ideal: Clear architecture, well-documented
   - Example: Score 90 = documented + clean deps; Score 40 = tangled mess

**Output**:

#### **Feature 2: Architecture Guardian**

**Description**: Real-time architectural enforcement with 12 commandments

**Commandments**:

1. **Respect Layer Boundaries** - UI â†’ Services â†’ Repositories â†’ Database
2. **No Circular Dependencies** - Module A â†’ B â†’ C âœ…, A â†’ B â†’ A âŒ
3. **Maintain High Cohesion** - LCOM4 < 0.5
4. **Minimize Coupling** - Depend on abstractions, not implementations
5. **Write Tests** - 80%+ coverage, every public function tested
6. **Document Public APIs** - JSDoc for all exports
7. **No Dead Code** - Remove unused imports, unreachable code, commented code
8. **Keep Functions Small** - Max 25 lines per function
9. **Use Meaningful Names** - No single letters (except i, j, k in loops)
10. **Handle Errors Gracefully** - Try-catch for all async operations
11. **Don't Repeat Yourself** - Max 5% code duplication
12. **Use Types** - TypeScript: explicit types on params + returns

**Violation Detection**:
- Real-time analysis on file save (< 100ms)
